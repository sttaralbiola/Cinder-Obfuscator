What happens in the forge

Constant encryption

Strings and numbers get pulled out, encoded, and rebuilt at runtime instead of sitting in the file as plain text.

Control-flow flattening

Your if/while/for structure is rewritten into one dispatch loop, so the logic no longer reads top-to-bottom.

Local renaming & proxying

Every variable name you wrote is gone — replaced with short, meaningless identifiers wired through proxy tables.

Use it from your own code

Same engine, no UI. POST your source, get back the forged result.

curl -X POST https://YOUR-APP.onrender.com/api/obfuscate \
  -H "Content-Type: application/json" \
  -d '{"code":"print(\"hi\")","preset":"Strong"}'

Returns { success, output, preset, bytesIn, bytesOut, ms }. GET /api/presets lists the available strength levels.

Based on Prometheus by Elias Oelschner — github.com/prometheus-lua/Prometheus
